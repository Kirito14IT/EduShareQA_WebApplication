## 项目背景与愿景
- **名称**：大学在线课程答疑系统（EduShareQA）  
- **愿景**：为高校学生和教师提供一个统一的学习资料共享与互动答疑平台，降低信息沟通成本、提升学习效率。  
- **目标用户**：上海理工大学光电信息与计算机工程学院的在校学生、课程教师，以及负责日常运维的系统管理员。  
- **核心价值**：打通“课程→资源→问答→通知”闭环，让学生能够快速找到所需资料、教师高效响应问题，管理员对平台内容进行治理。  

## 角色与职责
| 角色 | 来源 | 主要目标 | 约束 |
| --- | --- | --- | --- |
| 管理员 | 系统初始化创建 | 管理课程、教师、资源与问答内容，维护平台秩序 | 必须保障内容合规，支持紧急下线 |
| 教师 | 由管理员添加 | 发布权威资源、回答所属课程的问题、设置资源可见范围 | 仅能操作自己负责课程的数据 |
| 学生 | 自助注册 | 浏览/搜索/下载资源、上传学习资料、提问与跟进回答、管理个人内容 | 仅能管理自己上传的资源/问题 |

## 交付范围与优先级
### MVP（迭代 1，必须完成）
1. **账户体系**：学生注册登录、教师/管理员登录；通用的资料与密码修改。  
2. **课程与教师管理**（管理员）  
3. **学习资源模块**  
   - 学生/教师上传资源（单附件）、浏览、关键字搜索、分类过滤、下载计数  
   - 管理员内容审核（删除/修改说明）  
4. **问答模块**  
   - 学生按课程提问、上传图片附件、列表分页与搜索  
   - 教师查看所属课程的问题、作答（文本+图片）、编辑/删除回答  
   - 学生查看回答、问题状态提醒  
5. **通知中心（站内信）**：教师回答后提醒相关学生；教师看到未答问题提醒。  
6. **权限控制**：教师仅能访问自己课程；资源可见范围（公开/仅课程成员）。  

### 迭代 2（可延期）
- 资源多附件上传、批量导入课程/教师、课程公告、富文本版回答、资源收藏/点赞。  
- 更精细的通知（批量标记已读、消息筛选）、统计看板（活跃度、资源下载 Top）。  

## 需求明细
### 管理员模块
1. 课程管理：增删改课程名称、授课教师、课程描述、开课学院。  
2. 教师管理：增删改教师资料，配置多课程、简介、职称。  
3. 学习资源管理：查看全站资源，删除违规内容，编辑资源说明。  
4. 问答内容管理：删除或修改任意提问/回答。  

### 教师模块
1. 新问题提醒：展示“您所上课程有 X 条未回答问题”。  
2. 发布学习资源：上传资料（PDF/图片/压缩包），填写标题、说明、课程、可见范围。  
3. 回答问题：查看所属课程提问，发布/编辑/删除回答，支持图片附件。  
4. 资源可见范围：控制资源是否仅对本课程学生可见或对所有学生开放。  

### 学生模块
1. 通知提醒：教师回答后提醒“您有 X 条新回答”，可标记已读。  
2. 资源浏览：按课程过滤、按标题/简介关键字搜索，查看详情（上传者、简介、下载次数）。  
3. 资源上传：上传单个附件并填写标题、简介、所属课程。  
4. 问答区：在课程页面提问（标题、内容、图片附件），列表分页展示。  
5. 提问搜索：按课程/教师/关键字搜全站问答。  
6. 个人中心：查看/编辑/删除本人资源与提问，查看已被回答的问题。  

### 通用模块
- 个人资料：头像、邮箱等基本信息的查看与修改。  
- 密码修改：基于旧密码或重置邮件/验证码（后续可扩展）。  

## 非功能性需求
- **兼容性**：主流桌面浏览器（Chrome 111+、Edge 111+）。  
- **性能**：常规查询接口 P95 < 800ms；附件上传 ≤ 50MB。  
- **安全**：基于 JWT 的鉴权，前后端全程 HTTPS（部署阶段）；后台操作需审计日志。  
- **可维护性**：前后端分仓合作，接口通过 OpenAPI/Swagger 维护；所有功能由自动化测试覆盖关键流程。  
- **可扩展性**：文件存储初期本地，设计上预留 OSS/对象存储切换能力。  

## 假设与约束
- 交付时间灵活，但优先保证 MVP 范围质量。  
- 当前仅支持中文界面与内容，暂不考虑移动端适配。  
- 通知采用站内信，可结合 WebSocket 推送与回退轮询机制。  
- 部署初期运行在 Windows 环境，测试通过后容器化部署到云服务器。  

## 验收标准
1. 所有 MVP 功能在测试环境可操作，关键路径提供截图/录屏。  
2. 管理员可通过后台独立维护课程/教师/资源/问答。  
3. 学生与教师完成各自业务闭环（上传→浏览→通知→答疑）。  
4. 文档齐全：API 规范、数据库设计、运行手册。
